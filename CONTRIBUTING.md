# Contributing to KYC API Framework

Thank you for your interest in contributing. This framework exists to help community financial institutions — community banks, credit unions, and CDFIs — implement modern, inclusive KYC systems. Contributions that improve the framework's usefulness for those institutions are especially welcome.

---

## What We Welcome

**Documentation improvements**
- Clarifications to existing guides
- Additional use cases for institution types not yet covered
- Corrections to compliance mapping (regulations change — please flag outdated content)
- Additional examples for specific vendor integrations

**Architecture additions**
- New progressive verification path patterns
- Additional institution type configurations (e.g., CDFI, RIA, mortgage lender)
- Integration patterns for specific core banking systems (Symitar, FiServ, Jack Henry, etc.)

**Compliance updates**
- Updates when referenced regulations change
- New regulatory guidance that affects the framework
- State-level regulatory considerations

---

## How to Contribute

1. **Open an issue first** for significant additions or changes. This allows discussion before you invest time in writing.

2. **Fork the repository** and create a branch for your contribution.

3. **Write clearly.** This framework is read by compliance officers, BSA professionals, and engineers who are not necessarily experts in each other's domains. Plain language is preferred over jargon.

4. **Cite regulatory sources.** When making compliance-related claims, link to the primary source (CFR, Federal Register notice, agency guidance document).

5. **Submit a pull request** with a clear description of what you changed and why.

---

## What We Do Not Accept

- Proprietary or confidential implementation details from any employer or institution
- Vendor-specific content that reads as promotional rather than educational
- Contributions that weaken compliance guidance (e.g., suggesting ways to minimize screening requirements)
- Content that is not relevant to U.S. financial institution KYC

---

## Reporting Compliance Errors

If you find content that you believe is legally incorrect or could mislead a financial institution about their compliance obligations, please open an issue immediately with the tag `compliance-correction`. This is treated with high priority.

---

## Code of Conduct

Contributors are expected to engage professionally and constructively. The financial inclusion mission of this framework means we are especially mindful of perspectives from practitioners at smaller institutions and those who work directly with underserved communities.

---

## Questions

Open an issue with the `question` tag. Questions from practitioners at community institutions are welcome — your real-world experience helps improve the framework for everyone.
